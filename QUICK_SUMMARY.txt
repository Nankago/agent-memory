2. run_external_end_to_end COMMAND
════════════════════════════════════════════════════════════════════════════════

REQUIRED ARGUMENTS:
  --benchmark                 Benchmark name (e.g., "longmemeval", "locomo")
  --retrieval-variant         Retrieval variant (repeatable for batch runs)

INPUT OPTIONS (use ONE):
  --input-dir                 Single variant directory with retrieval_diagnostics.jsonl
  --input-root                Parent directory containing multiple variant folders

OPTIONAL ARGUMENTS:
  --method                    Method name (repeatable, defaults to all 3 methods)
  --model-dir                 Model artifacts (required for learned methods)
  --output-root               Output directory (default: outputs/external_runs)
  --summary-root              Summary table directory
  --top-k                     Cap on retrieved candidates
  --final-k                   Cap on reranked candidates

5. OUTPUT STRUCTURE
════════════════════════════════════════════════════════════════════════════════

Directory hierarchy:
  {output_root}/{benchmark}/{retrieval_variant}/{method}/
  ├── prediction_diagnostics.jsonl     Per-query prediction details
  ├── metrics.csv                       Single row with aggregate metrics
  └── metrics.json                      Same metrics in JSON format

Summary tables (written to summary_root):
  ├── external_end_to_end_metrics.csv   All metrics rows combined
  ├── external_end_to_end_summary.csv   Simplified projection
  └── external_end_to_end_summary.md    Markdown table for viewing

CSV columns per row:
  - benchmark, retrieval_variant, method, num_queries
  - accuracy (is_correct rate)
  - exact_match (normalized exact match)
  - hit_at_1, mrr (retrieval-only metrics)
  - retrieval_support_recall_at_retrieve_k
  - retrieval_support_recall_at_final_k
  - notes (method description)
